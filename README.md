## 🚀 Problem Statement

### The Challenge

Most people struggle with maintaining meaningful relationships in today's fast-paced world:

| Problem                            | Impact                                |
| ---------------------------------- | ------------------------------------- |
| ❌ Forgetting to check in          | Friendships drift over time           |
| 📅 Losing track of last contact    | Uncertainty about relationship status |
| 🤔 Not knowing who needs attention | Reactive instead of proactive         |
| 👀 No visibility into patterns     | Can't improve communication habits    |

### The Solution: KeenKeeper

KeenKeeper empowers you with:

✅ **Centralized Dashboard** - See all friendships at a glance  
✅ **Smart Tracking** - Know exactly when you last connected  
✅ **Visual Indicators** - Color-coded status shows who needs attention  
✅ **Easy Logging** - One-click interaction tracking  
✅ **Beautiful Analytics** - Visualize your communication habits

## 🔧 Configuration & Customization

### 🌍 Environment Variables

Create a `.env.local` file in the root directory for environment-specific settings:

```bash
# .env.local
NEXT_PUBLIC_API_URL=http://localhost:3000/api
NEXT_PUBLIC_APP_NAME=KeenKeeper
```

| Variable               | Purpose          | Default                     |
| ---------------------- | ---------------- | --------------------------- |
| `NEXT_PUBLIC_API_URL`  | API base URL     | `http://localhost:3000/api` |
| `NEXT_PUBLIC_APP_NAME` | Application name | `KeenKeeper`                |

### 🎨 Tailwind CSS & DaisyUI

DaisyUI is already configured in your global styles:

**File: `src/app/globals.css`**

```css
@import 'tailwindcss';
@plugin "daisyui";
```

To customize colors and themes, edit `tailwind.config.js`:

```js
module.exports = {
    daisyui: {
        themes: ['light', 'dark', 'cupcake'],
    },
};
```

### 🔄 Zustand Store

The state management is handled in `src/Lib/store.js`:

```javascript
import { create } from 'zustand';

export const useTimelineStore = create((set) => ({
    entries: [],
    addEntry: (entry) =>
        set((state) => ({
            entries: [entry, ...state.entries],
        })),
}));
```

### 📝 Commit Message Convention

We use conventional commits:

```
feat:     New feature
fix:      Bug fix
docs:     Documentation changes
style:    Code style changes (no logic change)
refactor: Code refactoring
test:     Adding/updating tests
chore:    Build, deps, tooling changes
```

**Examples:**

```bash
git commit -m "feat: add friend filtering by tags"
git commit -m "fix: correct status color calculation"
git commit -m "docs: update installation guide"
```

### ✅ Code Guidelines

- **Follow existing patterns** - Look at similar code for patterns
- **Keep it simple** - Prefer clarity over cleverness
- **Write comments** - Explain the "why" not the "what"
- **Test thoroughly** - Run `npm run lint` before submitting
- **Update docs** - If you change functionality, update README
